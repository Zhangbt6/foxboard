# 看板 / 流程图双主视图职责文档

> 维护人：黑狐 | 更新日期：2026-03-22
> 用途：明确 FoxBoard 两个主视图（看板 & 流程图）的职责边界与协作关系

---

## 一、总体定位

FoxBoard 有两个主视图：

| 视图 | 路由 | 渲染引擎 | 核心视角 |
|------|------|---------|---------|
| **看板 (Kanban)** | `/kanban` | React 组件 + CSS Grid | **任务状态**视角 |
| **流程图 (Workflow)** | `/workflow` | React Flow | **团队流程**视角 |

两者**数据同源**（均来自 FoxBoard 后端 API），但面向的使用场景不同。

---

## 二、看板视图职责

### 2.1 功能范围
- 按任务状态（TODOs / DOING / REVIEW / DONE / BLOCKED）分列展示
- 支持任务的创建、编辑、拖拽改状态
- 显示任务详情抽屉（TaskDrawer）
- 实时 WebSocket 推送任务状态变更
- 支持项目维度过滤（ProjectContext）

### 2.2 核心用户
- **花火姐姐**：任务派单、状态追踪
- **白狐 / 青狐**：领取任务、更新进度
- **黑狐**：REVIEW 审核、任务归档

### 2.3 数据来源
```
GET /api/kanban?project_id=xxx
GET /api/tasks/{task_id}
WS /ws → task_update 事件
```

### 2.4 不负责的事
- 不展示任务间的依赖关系（那是流程图的职责）
- 不展示团队工作流程（那是流程图的职责）
- 不展示全局统计（那是 Dashboard 的职责）

---

## 三、流程图视图职责

### 3.1 功能范围
- 展示团队工作流程的**线性步骤链**（需求提出 → 花火拆解 → 白狐调研 → 青狐开发 → 黑狐验证 → 花火汇总 → 主人审核）
- 每个步骤节点可关联任务，显示任务标题、优先级、执行人
- **FB-049**：依赖链/阻塞链可视化（边颜色区分：正常流 / 等待依赖 / 阻塞）
- **FB-053**：节点点击 → 打开任务详情抽屉（与看板联动）
- WebSocket 实时推送流程状态变更
- 支持多 Workflow 切换

### 3.2 核心用户
- **主人**：俯瞰团队全貌，了解流程卡点
- **花火姐姐**：确认各环节流转是否顺畅
- **黑狐**：确认审核环节是否有积压

### 3.3 数据来源
```
GET /api/workflows
GET /api/workflows/{workflow_id}/nodes
GET /api/tasks/{task_id}  (含 depends_on)
WS /ws → workflow_state_changed 事件
```

### 3.4 不负责的事
- 不做任务状态列管理（那是看板的职责）
- 不做任务创建/编辑（那是看板的职责）
- 不展示 Dashboard 指标图表

---

## 四、双视图协作规则

### 4.1 数据一致性
- 两视图数据均来自同一 FoxBoard 后端
- 任务状态变更通过 WebSocket 实时同步到两视图
- **禁止**在任一视图直接修改数据而不经过后端 API

### 4.2 联动机制（FB-053）
- 流程图节点点击 → 打开任务详情抽屉（TaskDrawer）
- 抽屉中更新任务状态 → 自动刷新两视图
- 看板中拖拽任务状态变更 → 流程图节点颜色同步变化

### 4.3 职责边界总结

| 场景 | 主导视图 |
|------|---------|
| 任务创建/编辑 | 看板 |
| 任务状态变更（拖拽） | 看板 |
| REVIEW 审核 | 看板（通过 TaskDrawer） |
| 依赖链/阻塞链查看 | 流程图 |
| 流程卡点定位 | 流程图 |
| 全局统计/图表 | Dashboard |
| 团队工作流全貌 | 流程图 |
| 任务分配/领取 | 看板 |
| 归档管理 | 看板（TaskDrawer 内） |
