## 2. 核心指标体系

### 2.1 任务吞吐量 (Throughput)

**定义**: 单位时间内完成并归档的任务数量

**计算公式**:
```
吞吐量 = 统计周期内状态变为 DONE 的任务数
```

**数据源映射**:
| 字段 | 来源表 | 字段名 | 条件 |
|------|--------|--------|------|
| 任务状态 | tasks | status | status = 'DONE' |
| 完成时间 | tasks | updated_at | 或单独 completed_at |

**可视化建议**:
- 折线图/柱状图
- 周期: 周/双周/月
- 支持按项目/负责人分组

---

### 2.2 任务完成周期 (Cycle Time)

**定义**: 任务从 DOING 状态到 DONE 状态的平均耗时

**计算公式**:
```
平均周期 = SUM(任务完成时间 - 任务开始时间) / 已完成任务数
单位: 天 (保留1位小数)
```

**数据源映射**:
| 字段 | 来源表 | 字段名 |
|------|--------|--------|
| 开始时间 | task_logs | created_at WHERE status='DOING' |
| 完成时间 | task_logs | created_at WHERE status='DONE' |
| 任务ID | task_logs | task_id |

**可视化建议**:
- 箱线图展示分布
- 平均值趋势线
- 按任务优先级/标签分组对比

---

### 2.3 任务在制品数量 (WIP - Work In Progress)

**定义**: 当前处于 DOING 或 REVIEW 状态的任务总数

**计算公式**:
```
WIP = 状态为 DOING 的任务数 + 状态为 REVIEW 的任务数
```

**数据源映射**:
| 字段 | 来源表 | 字段名 | 条件 |
|------|--------|--------|------|
| 任务状态 | tasks | status | IN ('DOING', 'REVIEW') |

**可视化建议**:
- 仪表盘/Gauge 图
- 显示当前值 vs 建议上限 (如 5-7 个任务/人)
- 趋势折线图

---

### 2.4 任务通过率 (Pass Rate)

**定义**: 任务一次提交即通过审核的比例

**计算公式**:
```
通过率 = (一次通过的任务数 / 总完成的任务数) × 100%
```

**数据源映射**:
| 字段 | 来源表 | 字段名 | 说明 |
|------|--------|--------|------|
| 打回次数 | task_logs | action='reject' 次数 | 按 task_id 分组统计 |
| 任务完成 | tasks | status='DONE' | 统计周期内 |

一次通过 = 该任务没有 reject 记录

**可视化建议**:
- 百分比仪表盘
- 趋势折线图
- 按项目/负责人分组对比

---

### 2.5 任务分布 (Task Distribution)

**定义**: 任务在各状态/优先级的分布情况

**计算公式**:
```
分布比例 = (某类别任务数 / 总任务数) × 100%
```

**数据源映射**:
| 维度 | 来源表 | 字段名 |
|------|--------|--------|
| 按状态 | tasks | status |
| 按优先级 | tasks | priority |
| 按标签 | tasks | tags |
| 按负责人 | tasks | assignee_id |

**可视化建议**:
- 饼图/环形图 (状态分布)
- 堆叠柱状图 (多维度交叉)
- 热力图 (负责人×优先级)
