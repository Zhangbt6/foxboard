# 任务规则文档 — FoxBoard AI 工作室

> 定义状态流转、Owner 规则、优先级定义、交付标准。
> 更新日期：2026-03-20

---

## 一、任务状态

| 状态 | 含义 | 谁改 |
|------|------|------|
| **TODO** | 任务已创建，待开始 | 花火写入 |
| **DOING** | 执行中 | 执行人认领 |
| **REVIEW** | 等黑狐验收 | 执行人完成后提交 |
| **DONE** | 黑狐通过 + 花火确认 | 花火标记 |
| **BLOCKED** | 阻塞，需花火介入 | 执行人/花火 |
| **MASTER-TODO** | 需主人执行 | 花火写入 |
| **MASTER-DONE** | 主人已完成 | 花火标记 |

---

## 二、状态流转

```
花火 TODO → 执行人 DOING → 执行人 REVIEW → 黑狐审核 PASS → 花火 DONE
                                              ↓ FAIL
                                         打回 DOING（修复后重新提交）

执行人 BLOCKED → 花火介入 → DOING
花火 MASTER-TODO → 主人完成 → MASTER-DONE
```

**关键规则**：
- DONE 前**必须**经过黑狐 REVIEW
- DONE 后禁止退回，修改走新任务
- 花火最终确认 DONE（黑狐 PASS 不等于 DONE，花火需二次确认）

---

## 三、辅助字段

| 字段 | 说明 |
|------|------|
| Owner | 第一责任人（唯一） |
| Priority | P1（紧急24h）/ P2（本周）/ P3（本月） |
| 类型 | 调研 / 开发 / 测试 / 文档 / 部署 |
| 协作者 | 协助人（非 Owner） |
| 依赖 | 前置任务 ID |
| 截止时间 | YYYY-MM-DD |

---

## 四、Owner 规则

1. 每个任务**必须只有一个 Owner**
2. Owner 负责推进任务状态
3. 协作者可多人，但不改变第一责任
4. Owner 不明确时，默认花火

---

## 五、交付标准

| 执行人 | 必须交付 |
|--------|---------|
| 白狐 | 调研报告 / 资源清单 / 对比表 / 推荐结论 |
| 青狐 | 代码 + 自测通过 / 功能说明 / 待处理问题列表 |
| 黑狐 | 测试记录 / 验收意见（PASS/FAIL + 修复建议） / 文档更新记录 |
| 花火 | 任务拆解 / 阶段汇报 / 看板状态 / 风险与建议 |

**没有交付物的任务不能标记 DONE。**

---

## 六、优先级定义

| 级别 | 含义 | 交付要求 |
|------|------|---------|
| P1 | 紧急，24小时内完成 | 即时执行 |
| P2 | 本周内完成 | 标准流程 |
| P3 | 本月内完成 | 正常排队 |
