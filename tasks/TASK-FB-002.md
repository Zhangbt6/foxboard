# TASK-FB-002: 竞品调研——开源项目管理看板

## 基本信息
- **状态**: TODO ⬜
- **版本**: v0.1.0
- **分配**: 白狐
- **优先级**: P2
- **创建**: 2026-03-20
- **依赖**: TASK-FB-001（PRD定稿）

## 目标
调研主流开源项目管理看板方案，为 FoxBoard 前端选型和功能设计提供参考。

## 输入
- FoxBoard PRD.md（花火提供）
- 竞品分析范围（白狐自行确定）

## 步骤
1. 搜集开源项目管理看板（重点关注：轻量、React、看板视图）
2. 分析以下维度：
   - 技术栈（React/Vue/Angular、数据库、实时能力）
   - 核心功能（看板、成员状态、流程图、通知）
   - 视觉风格（是否可定制像素风）
   - 接入成本（自托管难度、数据迁移）
   - 适合 FoxBoard 借鉴的点
3. 输出竞品对比表（至少 5 个方案）

## 输出
- `tasks/TASK-FB-002-report.md`：竞品调研报告
  - 方案列表（含 GitHub stars、最后更新时间）
  - 功能对比表
  - 技术栈对比
  - 推荐方案（最适合 FoxBoard 参考的 2 个）
  - 借鉴建议（哪些功能直接用、哪些需要自研）

## 推荐调研方案（白狐可增补）
1. Plane.so（React，开源自托管）
2. GitLab项目管理（已有使用习惯）
3. Metabase（数据看板参考）
4. KanbanTool 或 Zenkit（看板 UX 参考）
5. Star-Office-UI（像素风格直接参考）

## 审核标准
- 至少 5 个方案，覆盖不同技术栈
- 每个方案有 GitHub stars 和最后更新时间
- 有明确的"推荐 FoxBoard 参考"结论
