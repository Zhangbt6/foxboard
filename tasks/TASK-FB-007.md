# TASK-FB-007: 流程图点亮模块 PoC

## 基本信息
- **状态**: **已移至 Phase 3 (v0.3.0)** — 非 MVP，不在 Phase 1 执行
- **版本**: v0.3.0（原 v0.1.0）
- **分配**: 青狐
- **优先级**: P3
- **创建**: 2026-03-20
- **依赖**: TASK-FB-004（前端骨架）+ TASK-FB-005（状态协议）

## 目标
用 React Flow 实现 FoxBoard 项目流程图，展示任务节点的状态流转，阻塞节点红色高亮。

## 输入
- 前端骨架（TASK-FB-004）
- Agent 状态协议（TASK-FB-005）
- PRD.md v1.1（流程图设计）

## 步骤
1. 安装 React Flow：
   ```bash
   cd frontend
   npm install @xyflow/react
   ```
2. 设计 FoxBoard 项目标准流程节点：
   ```
   需求提出 → 花火拆解 → 白狐调研 → 青狐开发 → 黑狐验证 → 花火汇总 → 主人审核
   ```
3. 实现 FlowChart 组件：
   - 从 FoxBoard 后端读取任务状态
   - 根据任务状态决定节点颜色（灰/蓝/绿/黄/红）
   - 阻塞节点（REVIEW 超过阈值）红色高亮 + 闪烁动画
4. 创建流程图页面 `/flows/:projectId`
5. 接入后端 API，实现节点状态实时刷新

## 节点状态样式
| 状态 | 颜色 | 样式 |
|------|------|------|
| 未开始 | 灰色 | 默认 |
| 进行中 | 蓝色 | 边框发光 |
| 完成 | 绿色 | ✅ 图标 |
| 等待审核 | 黄色 | ⚠️ 图标 |
| 阻塞 | 红色 | 闪烁动画 |

## 输出
- `frontend/src/components/FlowChart.tsx`：流程图组件
- `frontend/src/pages/FlowPage.tsx`：流程图页面
- 集成到前端路由

## 审核标准
- 流程图正常渲染所有节点
- 节点颜色与任务状态一致
- 阻塞节点有明显的红色高亮效果
- 流程图数据从后端 API 读取
